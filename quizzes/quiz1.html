Java OOP Quiz: Exclusively for Your Placement Online Assessments and Interviews

Q1. Which of the following is NOT a pillar of Object-Oriented Programming (OOP) in Java?

Answer: d) Compilation

OOP is based on four main pillars: Inheritance, Encapsulation, Polymorphism, and Abstraction. Compilation is not a concept of OOP but rather a process of converting code into machine-readable format.

Q2. In Java, what is the purpose of the super keyword?

Answer: b) It is used to invoke the superclass constructor or methods.

The super keyword in Java is used to refer to the immediate parent class object. It is commonly used to call superclass methods or constructors when you need to override them or initialize a subclass with a specific constructor from the parent class.

Q3. Which of the following best describes polymorphism in Java?

Answer: b) Defining multiple methods with the same name but different signatures.

Polymorphism allows one interface to be used for a general class of actions. In Java, method overloading (compile-time polymorphism) and method overriding (run-time polymorphism) are examples. This means multiple methods can have the same name but differ by parameters (overloading) or behavior (overriding).

Q4. Which one of the following are essential features of an object-oriented programming language?

Answer: d) Abstraction, encapsulation, inheritance, and polymorphism

The four essential features of an object-oriented programming language are abstraction, encapsulation, inheritance, and polymorphism. These concepts help in code modularity, reusability, and the efficient organization of large software systems.

Q5. Which of the following languages supports polymorphism but not classes?

Answer: c) Ada

Ada supports polymorphism through its tagged types and subprogram dispatching but does not follow a typical class-based object-oriented approach as seen in Java or C++.

Q6. Which feature of OOP indicates code reusability?

Answer: d) Inheritance

Inheritance allows a class to inherit properties and methods from another class, which encourages code reusability. This is a core principle of OOP that helps in reducing code duplication and improving maintainability.

Q7. Which of the following is associated with objects in Java?

Answer: d) All of the above

In Java, objects are characterized by their state (attributes), behavior (methods), and identity (unique reference or address). These three aspects define an object and its interaction within the program.

Q8. Why is Java not considered a purely Object-Oriented language?

Answer: d) All of the above

Java is not purely object-oriented because it supports primitive data types, uses the static keyword, and involves wrapper classes that convert primitives to objects (autoboxing and unboxing). These factors prevent Java from being fully object-oriented, unlike languages such as Smalltalk.

Q9. Why is Java not considered a purely Object-Oriented language despite having Wrapper Classes?

Answer: b) Wrapper classes internally use primitive data types during operations.

Even though Java uses wrapper classes to represent primitive data types as objects, during operations like arithmetic, Java still internally uses primitive types. This violates the pure object-oriented principle where all operations must be performed through objects.

Q10. What is the primary purpose of a class in Java?

Answer: b) To represent a blueprint or prototype for creating objects.

A class in Java serves as a blueprint from which objects are created, encapsulating both data members and methods that define the behavior of the objects.

Q11. Which of the following components can be included in a class declaration in Java?

Answer: d) All of the above

A class in Java can include data members, methods, constructors, and even nested classes and interfaces, making it a versatile structure for modeling real-world entities.

Q12. In the context of classes in Java, what is a constructor?

Answer: b) A special method used to initialize objects.

A constructor is a special method in Java that is called when an object is instantiated, allowing the object to be initialized with specific values.

Q13. What does the keyword this refer to in a constructor in Java?

Answer: c) It refers to the current object being created.

In a constructor, this is a reference to the current object, allowing access to its attributes and methods.

Q31. What is encapsulation in Java?

Q37. Which type of polymorphism is achieved through method overriding?

Runtime polymorphism occurs when a method call is resolved at runtime, typically through method overriding where a subclass provides a specific implementation of a method defined in its parent class.

Q39. What will be the output of the following code?

    }

    void sound() {

    } 
} 
class Main { 
    public static void main(String[] args) {

    }

Because of runtime polymorphism, the method of the subclass (Dog) is called at runtime, resulting in "Dog barks" being printed.

Q44. What is the main purpose of an interface in Java?

Q46. Which keyword is used to implement an interface in a class?

Q63. How does the this keyword help to differentiate between local and instance variables within a class? Provide an example.

Answer: b) By using the this keyword to access instance variables

The this keyword allows access to instance variables when local variables have the same name. For example:

class Example { int a; // Instance variable
Example(int a) { // Constructor parameter
    this.a = a; // 'this.a' refers to the instance variable
}
}

In this code, this.a differentiates the instance variable a from the parameter a of the constructor.

Q64. In the context of constructors, how is this() used to invoke another constructor within the same class? Explain with a code example.

Answer: c) To invoke another constructor

The this() keyword is used to invoke another constructor in the same class, allowing for constructor chaining. For example:

class Test { int a, b;
Test() {
    this(10, 20); // Calls the parameterized constructor
}

Test(int a, int b) {
    this.a = a;
    this.b = b;
}
}

In this example, the default constructor uses this(10, 20) to call the parameterized constructor, initializing the instance variables.

Q67. In the provided examples, how is the this keyword utilized to return the current class instance from a method? Explain with an example.

Answer: c) By using this to refer to the current instance

The this keyword can be used to return the current class instance from a method. For example:

class Test { int a, b;
Test() {
    a = 10;
    b = 20;
}

Test get() { return this; } // Returns the current instance

void display() {
    System.out.println("a = " + a + ", b = " + b);
}
}

public class Main { public static void main(String[] args) { Test obj = new Test(); obj.get().display(); // Calls display on the returned instance } }

In this example, the get() method returns the current instance, allowing display() to be called on it.

The this keyword in Java is used to refer to the current object instance.
